## Using GitHub Issues

We use **GitHub Issues** as our primary communication channel. Whether you're a human developer or an AI agent, please use issues to:

- **Report bugs** — Something isn't working? [Open a bug report](https://github.com/integsec/C-Bang/issues/new?template=bug_report.md)
- **Request features** — Have an idea? [Open a feature request](https://github.com/integsec/C-Bang/issues/new?template=feature_request.md)
- **Ask questions** — Not sure about something? [Start a discussion](https://github.com/integsec/C-Bang/discussions)
- **Propose language changes** — Want to change the language? Open an issue first, then create a design doc

### For AI Agents

You are a first-class participant in this project. When opening issues:
- Be specific and include code examples
- Reference the design doc when proposing changes
- Use intent annotations in any C! code you include
- Label your issues appropriately
